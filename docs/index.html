Docs

Everything on this page is included: the standard library is plain Scheme (lib/prelude.ss, auto-loaded at startup), and the rest is built into the VM.

The language

JetScheme is a compact dialect of Scheme in the R7RS-small tradition: lexical scope, first-class functions, and proper tail calls, so deep tail recursion doesn't grow the stack. The forms are quote, define, lambda, if, begin, let, let*, letrec, letrec* (plus named let), set!, and, or, cond with else, when, unless, and include, which includes another source file. apply covers calls whose argument list isn't known until runtime. Macros, quasiquote, and continuations aren't implemented yet.

(define (make-counter)
  (let ((n 0))
    (lambda ()
      (set! n (+ n 1))
      n)))

(define a (make-counter))
(define b (make-counter))

(displayn (list (a) (a) (b)))   ; prints (1 2 1) — b kept its own n

All numbers are 64-bit doubles. There's no exact/inexact split; exact? and integer? are #t for integral doubles. Strings and characters are byte-level UTF-8. string-length, string-ref, and string-set! are byte-indexed; bytewise comparison still yields correct codepoint order on valid UTF-8. Case conversion and character classes are ASCII-only.

Numbers

+ - * /, quotient, remainder, modulo, abs, square, expt, sqrt, exp, log, sin, cos, tan, asin, acos, atan, floor, ceiling, truncate, round, min, max, and random, which takes no arguments (random-seed reseeds it from the OS).

Comparisons = < <= > >= and predicates zero?, positive?, negative?, even?, odd?, number?. Convert with number->string and string->number. Bitwise ops — bitwise-and, bitwise-ior, bitwise-xor, bitwise-not, arithmetic-shift — operate on integral doubles.

Pairs & lists

cons, car, cdr, set-car!, set-cdr!, pair?, null?, list, list?, the caar…cddddr combinations, and the named accessors first through fifth and rest.

List operations: length, append, reverse, take, drop, last, concatenate, map, for-each, filter, fold, fold-right, reduce-right, and the association-list lookups assq, assv, assoc.

(define (sum xs)
  (let loop ((xs xs) (total 0))     ; named let — loops via tail call
    (if (null? xs)
        total
        (loop (cdr xs) (+ total (car xs))))))

(displayn (sum '(1 2 3 4 5)))               ; prints 15

Strings & characters

string, make-string, string-append, substring, string-copy, string-copy!, string-fill!, string-length; indexing and mutation go through the generic ref / setf! described below. Comparisons come in case-sensitive (string=?, string<?, …) and case-insensitive (string-ci=?, …) variants. Beyond comparisons there are string-upcase, string-downcase, string-for-each, string-map, and conversions between strings, lists, and vectors (string->list, list->string, string->vector, vector->string).

Characters: char->integer, integer->char, comparisons (char=?, char-ci=?, …), predicates char-alphabetic?, char-numeric?, char-whitespace?, char-upper-case?, char-lower-case?, plus char-upcase, char-downcase, and digit-value. Named literals: #\space, #\newline, #\tab, #\return, #\null, #\escape, #\backspace, #\delete.

Vectors & bytevectors

vector, make-vector, vector?, vector-length; bytevector, make-bytevector, bytevector?, bytevector-length, bytevector-copy, bytevector-copy!, bytevector-append. Both are read and written through ref / setf!; vector-ref, vector-set!, bytevector-u8-ref, and bytevector-u8-set! are also available.

Structs

struct, isa?. Each (struct ...) call yields a distinct type, even when invoked twice with the same name and field list.

(define point (struct 'point '(x y)))            ; create a fresh type
(define p (point 3 4))                           ; call the type to construct
(isa? p point)                                   ; => #t
(ref p 'x)                                       ; => 3
(setf! p 'x 10)                                  ; mutate field

Tuples

tuple. Constructs an immutable, fixed-size tuple. Tuples use value-based equal?, identity-based eq?, and numeric ref access.

(define t (tuple 'a 'b 'c))
(isa? t tuple)                                   ; => #t
(ref t 1)                                        ; => b

Generic access

(ref v i) is a polymorphic accessor. It works on vectors, strings, and tuples (integer index) and ordinary structs (symbol field name).

(setf! v i x) stores x at key i in v:

(setf! v 3 'x)                                   ; vector-set!
(setf! s 0 #\H)                                  ; string-set!
(setf! pt 'x 99)                                 ; struct field mutation

Equality & symbols

eq? compares identity, eqv? identity-or-number, and equal? compares structures recursively. There are also type-specific predicates boolean=?, symbol=?, and string=?. Symbols convert with symbol->string and string->symbol; symbol? and procedure? test for symbols and procedures.

I/O & files

display prints human-readable and write prints machine-readable, both to stdout; newline prints a newline, and displayn is display followed by a newline. read parses one datum at a time from an input port:

(define p (open-input-file "data.ss"))
(define a (read p))                                ; first datum
(define b (read p))                                ; second datum
(close-input-port p)

For character-level file work there are open-input-file, open-output-file, read-char, write-char, close-input-port, close-output-port, plus the predicates input-port?, output-port?, and eof-object?.

Running programs

argv is a vector of the script's command-line arguments, with the binary and script path already stripped; exit ends the process. The prelude auto-loads from $JET_PRELUDE/lib/prelude.ss — set JET_PRELUDE to use a different one, or pass --no-prelude to skip it.

Besides jet run (or just jet file.ss) the binary provides jet compile (source → bytecode on stdout), jet exec (run pre-compiled bytecode), and jet disasm (disassemble source or bytecode). --no-inline, --no-lift-lambdas, and --no-specialize-ops switch off individual optimizations; debug builds add --trace for an opcode-by-opcode execution trace.

The compiler also understands ($check expr): if expr is false it reports the source position and fails. $file, $line, and $col expose that position directly.

Impl

JetScheme is built on techniques described in:

  • Tail-threaded dispatch (Bell, 1973) — each opcode is its own small function; each function ends by tail-calling the next opcode's function directly. There's no central dispatch loop and no callee-save spills between opcodes; hot VM state stays in registers across the whole run.
  • Inline caching (Deutsch & Schiffman, 1984) — on the first time through a call site, the slow path patches the opcode in place to point at a specialized fast version of itself. Subsequent passes through the same site skip the lookup, key compare, and type check entirely.
  • Flat closures (Dybvig, 1987) — captures live in an inline array on the closure; immutable captures by value, mutated and shared ones get a heap indirection.
  • A-normal form (Flanagan et al., 1993) — every operand is a variable or a literal and every intermediate result gets an explicit name, so the later passes (inlining, register allocation, instruction selection) rewrite one flat shape instead of arbitrary nested trees.
  • NaN-boxing (Gudeman, 1993) — every value fits in 8 bytes: numbers are raw doubles; immediates (bool, char, empty list) and tagged heap pointers all hide in the IEEE 754 quiet-NaN payload. Type tests are cheap bit ops and any value passes in a single register.
  • Superinstructions (Proebsting, 1995) — common back-to-back opcode pairs are emitted as a single fused opcode: one chain step instead of two, and the body can skip the intermediate register move.
  • Opcode replication for ICs (Ertl & Gregg, 2003) — each cached opcode has several near-identical copies that rotate per call site, so distinct sites land on distinct branch-target-predictor entries instead of all colliding on one.
